Things I Wish I Knew Before Building a Mobile App

2019-06-19

Here’s the Gist…

A couple of months ago, I built an app called ‘Russ’ using React Native. Russ is a free to download iOS app which aims to raise awareness about climate change.

From a technical perspective, it was pretty crap when I released the first version. Over time, as I became more familiar with mobile app development I started to pinch myself about things I should’ve done way before I released the first version in order to make the app more kickass.

Here’s what I wish I knew and what you’re about to know…

Don’t Build Games in React Native

Ok so first and foremost…

Building games in React Native is really difficult

As RN is optimised for building applications which are heavily UI based, looking back at it, building Russ using it was not the best option as I made a lot of extra work for myself in replicating parts of physics engines that I needed and in general, ended up spending a lot more time on certain things than I should’ve.

By all means, it’s totally possible to build games in RN it’s just going to be a little bit of extra work. Personal opinion: I think it’s better to use a game engine such as Unity, Unreal or Corona. There are libraries such as Matter.js and React Native Game Engine to assist you in making games with RN but I still believe that a more productive way to utilise time will be through the use of a game engine.

Work in a Team

This seems like a pretty general thing to do for any project. If you’re working on an app to kill off some spare time, work in a team. Hit up friends who are keen on helping you with your app as it can:

  • Save you time
  • Improve your app in multiple areas (concept, graphics, software, etc)
  • The more brains, the better
  • More fun

Working alone does have it’s benefits as well, however, if I could backtrack to when I was working on Russ, I would’ve definitely asked some mates who were interested in the app idea to come onboard the ride and work on it with me.

Be Prepared to Pay

The moment I finished building Russ, I was literally over the moon and I probably did something really stupid thanks to the rush of endorphins.

That was until I realised the membership fee for Apple’s development program.

If you’re building apps for Android or iOS, just make sure you have cash in your account but apart from that, memberships shouldn’t stop you from app making awesomeness.

Take Breaks

Building an app can be immensly frustrating…

Actually building any software application can have times when everything just sucks.

So take breaks. This is something I definitely didn’t do enough and it took a bit of a toll on my health as I fell into the trap of becoming addicted to working on my app. If I had taken more breaks, I probably would’ve been more efficient and less stressed.

When the going gets tough and you just can’t find that impenetrable bug, step back from the screen, eat some food, watch some Netflix and then get back to work with a fresh axe.

Plan a Lot

I feel like this was something I executed fairly well when I was building Russ but I could’ve done a bit better.

Plan your app to the finest detail. Draw storyboards, write out daily tasks, do some daily reflection and don’t ignore bugs (because they will in most occasions, come back to bite you in the bum). By planning out an app and keeping track of it’s development progress, it becomes way easier to build it efficiently.

That’s That

I’m definitely not an experienced mobile apps developer so overtime, I’ll probably keep posting about more useful things to keep in mind. But, if I could invent a time machine and go back to when I was making Russ, these are definitely things I’d tell myself to keep in mind.

Guess you live and learn?

Good luck with your app adventures!
